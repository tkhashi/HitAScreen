# HitAScreen 残り実装計画（設定UI拡張中心）

## 1. 目的
`requiement.md` の In Scope を満たすため、現状実装との差分を埋める。
本計画は残り実装を対象とし、OCR/画像ベース検出は将来対応とする。

## 2. スコープ
- 対象: FR-1, FR-2, FR-3, FR-4, FR-5.1, FR-6, FR-7, FR-8, FR-10, FR-11 と関連 NFR
- 対象外（将来対応）:
  - OCR 機能
  - 画像ベース検出（FR-5.2）
  - GitHub Packages への配布

## 3. 実装フェーズ（実行順）

### Phase 1: 設定UI拡張（最優先）
- 1-1. 各種ショートカットキー設定
  - 起動ホットキー + 操作キー（モニタ左右切替/再解析/アクション切替）を設定可能化
  - `ESC/Enter/Backspace` は固定キーとして維持
- 1-2. AX取得フィルター設定
  - 除外Role方式で設定可能化（例: `AXGroup`）
  - 除外Role設定を候補抽出ロジックへ適用
- 1-3. ラベル外観設定
  - 色・透明度・ラベルサイズ・ラベルフォントサイズを設定可能化
  - 保存後にオーバーレイへ即時反映
- 1-4. OS許可状態表示 + 設定導線
  - Accessibility / Input Monitoring / Screen Recording の状態表示
  - 権限別に OS 設定画面を開くボタンを提供
- 成果物:
  - 設定画面から4項目を操作でき、再起動後も保持される

### Phase 2: 要件ギャップ残件
- 2-1. ホットキー競合検出（FR-2.2）
- 2-2. OS起動時自動起動 ON/OFF（FR-1.2）
- 2-3. 診断情報の拡張（FR-11）
- 成果物:
  - FR-1.2 / FR-2.2 / FR-11 の受け入れ確認

### Phase 3: 品質保証
- 3-1. 自動テスト追加
  - 設定保存/読込の互換
  - ショートカット解決
  - AX除外Roleフィルター
  - 外観設定値のバリデーション
- 3-2. 手動テスト実施
  - OS依存挙動（グローバルホットキー、権限導線、実アプリ反映、複数モニタ）
- 3-3. NFR計測
  - 表示遅延、長時間入力時の遅延傾向、メモリ傾向
- 成果物:
  - 手動テスト結果記録（`docs/manual-tests/MANUAL_TEST_RESULT_TEMPLATE.md`）

### Phase 4: 後回しタスク（最後に着手）
以下は必ず最後に実装する。
- 4-1. ラベル生成/確定方式の改修（prefix衝突を設計上排除）
  - ラベル長は同一長で生成する
  - prefix衝突（例: `A` と `AB`）を生成しない
  - 入力途中は候補フィルタのみ行い、確定は完全一致時のみ実行する
  - 文字割当は home row 優先（例: `ASDFGHJKL`）で、balanced hint generation を適用する
- 4-2. 初回ホットキー時の縦位置ズレ
- 4-3. 2回目以降のヘルプテキスト隠れ
- 4-4. ラベル重なり対策

### Phase 5: 配布準備（GitHub Release）
- 5-1. 配布方針を GitHub Releases のみに固定（Packages は利用しない）
- 5-2. `self-contained` / `osx-arm64` の publish 導線を CI に追加
- 5-3. `.app` 組み立て（`Info.plist` / `AppIcon.icns` / 実行バイナリ配置）
- 5-4. `hit-a-screen-icon.png` 起点の `icns` 生成（必要に応じて単純化）
- 5-5. `codesign`（entitlements 含む）/ `notarize` / `staple`
- 5-6. 配布物生成（`dmg` + `SHA256SUMS.txt`）
- 5-7. Git tag（`vX.Y.Z`）トリガーで Release へ資産アップロード
- 5-8. README に導入手順・権限付与手順を追記
- 成果物:
  - Release から `dmg` を取得し、.NET 未導入環境でも起動できる

## 4. 設計変更点（公開IF/設定モデル）
- `UserSettings` に以下を追加する。
  - ショートカット設定（起動 + 操作キー）
  - AX除外Role設定
  - ラベル外観設定（色/透明度/サイズ/フォントサイズ）
- `IPermissionService` に権限別設定画面を開く API を追加する。
- ショートカット判定のハードコードを設定参照へ置換する。
- ラベル生成/確定ロジックは以下に統一する。
  - 固定長ラベル生成
  - prefix衝突なし
  - 途中入力はフィルタのみ（実行しない）
  - 完全一致でのみ実行
  - home row 優先 + balanced hint generation

## 5. 受け入れ基準（残り実装向け）
- AC-R1: 設定画面でショートカット/AXフィルター/外観/許可導線を操作できる
- AC-R2: 設定値が再起動後も維持される
- AC-R3: 権限不足でもクラッシュせず、状態表示と設定導線が機能する
- AC-R4: ホットキー競合時にユーザーへ警告できる
- AC-R5: 自動起動設定が ON/OFF で反映される
- AC-R6: Phase 4 完了後、MT-10〜MT-13 が合格する
- AC-R7: `A` / `AB` のような prefix衝突ラベルが生成されず、2文字以上ラベルを追加操作なしで選択できる
- AC-R8: GitHub Release に `HitAScreen-<version>-macos-arm64.dmg` と `SHA256SUMS.txt` が公開される
- AC-R9: クリーン環境で「破損/不完全」警告なく起動でき、権限付与後に基本操作が成立する
