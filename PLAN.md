# HitAScreen 残り実装計画

## 1. 目的
`requiement.md` の In Scope を満たすため、現状実装との差分を埋める。
本計画は「残り実装」を対象とし、既存の実装済み機能は再設計しない。

## 2. スコープ
- 対象: FR-1, FR-2, FR-3, FR-4, FR-5.1, FR-6, FR-7, FR-8, FR-10, FR-11 と関連 NFR
- 対象外（将来対応）:
  - OCR 機能
  - 画像ベース検出（FR-5.2）

## 3. 現状整理（要約）
- 実装済み（概ね）:
  - 常駐 + トレイ起動
  - グローバルホットキー起動
  - アクティブウィンドウ情報取得とフォールバック
  - Accessibility ベース候補抽出（FR-5.1）
  - ラベル表示、左/右/ダブル/フォーカス操作
  - 複数モニタ切替、再解析
  - 設定保存、診断表示
- 未完了/改善必要:
  - ホットキー競合検出（FR-2.2）
  - 自動起動設定（FR-1.2）
  - 安定性/性能要件の定量検証（NFR-1/2/3）
  - 既知不具合（`docs/TODO_CRITICAL.md`）

## 4. 実装フェーズ（実行順）

### Phase A: 安定化（最優先）
- A-1. 終了時チラつき調査と修正
- A-2. 入力遅延の再現条件固定、計測ログ追加、根本原因修正
- A-3. クリック信頼性の最終化（必要なら `AXPress` フォールバック追加）
- 成果物:
  - 再現手順 + 修正内容 + 手動テスト結果

### Phase B: 要件ギャップの実装
- B-1. ホットキー競合検出（設定時に警告表示）
- B-2. OS 起動時自動起動 ON/OFF 実装
- B-3. 診断情報の拡張（取得項目・処理時間・候補数の見やすさ改善）
- B-4. 設定UIの不足項目整理（抑制条件、表示制御）
- 成果物:
  - FR-1.2 / FR-2.2 / FR-11 の受け入れ確認

### Phase C: 品質保証
- C-1. 主要 Core ロジックの単体テスト追加
  - ラベル確定
  - 抑制判定
  - 再解析/モニタ切替
  - フォールバック分岐
- C-2. 手動テスト計画 MT-01〜MT-09 実施
- C-3. NFR 観点の測定（表示遅延、長時間入力遅延、メモリ傾向）
- 成果物:
  - テスト結果記録（`docs/manual-tests/MANUAL_TEST_RESULT_TEMPLATE.md`）

### Phase D: 後回しタスク（最後に着手）
以下は必ず最後に実装する。
- D-1. 複数文字ラベル確定不具合
- D-2. 初回ホットキー時の縦位置ズレ
- D-3. 2回目以降のヘルプテキスト隠れ
- D-4. ラベル重なり対策 + 候補表示フィルタ設定

## 5. 受け入れ基準（残り実装向け）
- AC-R1: 他アプリへの長時間入力で遅延が増加しない
- AC-R2: 終了時にフォーカスちらつきが発生しない
- AC-R3: Finder / ブラウザ / VS Code / Electron 系でクリック操作が安定
- AC-R4: ホットキー競合時にユーザーへ警告できる
- AC-R5: 自動起動設定が ON/OFF で反映される
- AC-R6: D-1〜D-4 を満たした後、MT-10〜MT-13 が合格

## 6. リスクと対策
- EventTap の負荷増大:
  - コールバック内処理を最小化し、重い処理は UI スレッド/ワーカーへ移譲
- AX 要素の重複/過多:
  - Role/階層/親子距離で候補フィルタ
- 座標ズレ:
  - 初回描画時に display bounds と safe-top を再同期

## 7. 完了定義
以下を満たした時点で本計画は完了。
- Phase A〜D 全タスクの実装完了
- 自動テストが通過
- 手動テスト MT-01〜MT-13 で重大不具合なし
- `README.md` / `docs/` が実装状態と一致
